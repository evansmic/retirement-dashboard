# Sprint 644: Anonymization Checklist

## Purpose
Prepare an outside-app checklist that keeps feedback review private and local-first.

## Checklist
- Use built-in examples when possible.
- If a reviewer uses their own plan, they keep the file on their own device.
- Do not ask for names, addresses, account numbers, employer names, exact institution names, or uploaded plan files.
- Record only summarized reactions and anonymized issue notes.
- Round sensitive dollar amounts in any manual note unless exact values are needed to reproduce a product issue.

## Outcome
- Manual feedback can be summarized without collecting private plan files.
- Local-first trust remains visible during review.

## Boundary
This sprint does not create a repository of plan files, screenshots with personal details, or persistent app feedback records.
