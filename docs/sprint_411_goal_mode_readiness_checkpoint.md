# Sprint 411 - Goal Mode Readiness Checkpoint

## Decision

Evidence wording is stable enough to continue toward goal-mode re-ranking prep, but goal modes themselves remain deferred.

## Rationale

The richer example slice keeps the first-review order consistent across pension-heavy, early-retirement, and already-retired household stories. That supports the next architecture batch: planning how the same bounded candidate set could be re-ranked for alternate goals while keeping the normal UI focused on the recommended plan.

## Next likely direction

- Define goal-mode scoring labels for max spend, max estate, and lower tax.
- Prove goal modes re-rank the same candidate set only.
- Keep goal-mode research deferred from the normal UI until feedback says it helps normal users.
