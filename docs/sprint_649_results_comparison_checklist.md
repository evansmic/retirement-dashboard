# Sprint 649: Results Comparison Checklist

## Purpose
Give manual reviewers a consistent path through Results without requiring a UI redesign.

## Checklist
- Start on Overview and ask whether the retirement answer is visible.
- Check whether monthly spending is understandable and framed calmly.
- Check whether top review actions feel like review items, not instructions.
- Open Details and locate broad withdrawal-family evidence when present.
- Compare current plan framing with broad-family framing.
- Note whether tax, OAS recovery, funded years, and money-left evidence are understandable.
- Confirm whether the local save and export boundaries feel private.

## Outcome
- Review sessions produce comparable notes across examples and local plans.
- The checklist stays compatible with the current UI.

## Boundary
This sprint does not start the UI overhaul, add new panels, or alter consumer workflow.
