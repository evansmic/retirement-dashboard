# Sprint 648: Confusion Signal Triage

## Purpose
Classify reviewer confusion without turning every comment into product scope.

## Triage
- **Copy cleanup:** reviewer understands the concept after a wording change.
- **Input clarity:** reviewer cannot tell which input drives the result.
- **Details organization:** reviewer cannot find the evidence or loses context.
- **Evidence gap:** reviewer needs a comparison metric that is already available but not clear.
- **Deferred scope:** reviewer asks for annual account instructions, advisor workflow, tax filing detail, or cloud sharing.

## Outcome
- Confusion can guide small fixes without starting annual sequencing.
- Deferred-scope requests are preserved as signals, not immediate implementation tickets.

## Boundary
This sprint does not widen optimizer search, add annual instructions, or change the engine.
