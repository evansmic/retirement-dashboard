# Sprint 400 - Annual Sequencing Prerequisite Register

## Purpose

List what should be true before planning annual account-level withdrawal sequencing.

## Prerequisites

- More real-user feedback on whether the recommended plan and spending number feel trustworthy.
- Goal-mode review that can re-rank the same bounded candidates for max spend, max estate, or lower tax without expanding the normal UI.
- Flexibility and cash-wedge review language that users understand as context, not instructions.
- Better explainability for why a broad withdrawal family was selected.
- Performance confidence in the React results workspace before adding heavier search.
- Clear province and edge-case strategy before making account sequencing more granular.

## Boundary

Annual account-level sequencing remains deferred. This register is a planning guard, not an implementation plan.
